## 1. 專案概覽

這是一款以 **Godot 4** 開發的三消寶石戰鬥遊戲。玩家透過在棋盤上點擊相連的同色寶石來消除它們，觸發角色攻擊敵人。遊戲結合了：

- **三消消除**：點擊即消（非交換），相連同色寶石 ≥ 最小數量即可消除
- **RPG 戰鬥**：角色有攻擊力、血量、技能、元素屬性
- **元素克制**：火→葉→水→火 的三角克制（1.5 倍傷害）
- **技能系統**：被動、主動、回應三種技能類型
- **高階寶石**：融合達標可生成火球/火柱等特殊寶石

### 檔案結構

```
scripts/
  ├── game_state.gd       # 全域遊戲狀態（Autoload 單例）
  ├── stage_data.gd        # 關卡資料定義
  ├── character_data.gd    # 角色資料定義
  ├── enemy_data.gd        # 敵人資料定義
  ├── main.gd              # 主場景控制器（核心協調者）
  ├── board.gd             # 棋盤邏輯（消除、掉落、融合提示）
  ├── block.gd             # 單個寶石方塊
  ├── battle_manager.gd    # 戰鬥管理器（回合、傷害、敵人）
  ├── enemy.gd             # 敵人 UI 控制
  ├── character_panel.gd   # 角色面板（卡片顯示）
  ├── character_detail.gd  # 角色詳細資訊畫面
  ├── characters_screen.gd # 角色列表畫面
  ├── map.gd               # 地圖/關卡選擇畫面
  ├── projectile.gd        # 拋物線彈道特效
  ├── gem_particle.gd      # 寶石粒子特效
  ├── slash_effect.gd      # 斬擊特效
  ├── damage_number.gd     # 浮動傷害數字
  └── bullet_projectile.gd # 子彈彈道特效
```

## 3. 資料模型

### GameState（全域狀態）

全域 Autoload 單例，跨場景保持資料：

| 屬性 | 說明 |
|------|------|
| `selected_stage` | 玩家選擇的關卡 |
| `selected_party` | 玩家選擇的隊伍（最多 4 人） |
| `owned_characters` | 擁有的所有角色 |
| `detail_character` | 要查看詳細資訊的角色 |

### StageData（關卡資料）

| 屬性 | 說明 |
|------|------|
| `stage_name` | 關卡名稱 |
| `columns / rows` | 棋盤大小 |
| `min_match` | 最少連接數（預設 2） |
| `allowed_types` | 允許出現的寶石類型 |
| `rounds` | 敵人波次陣列（`Array[Array[EnemyData]]`） |

### CharacterData（角色資料）

| 屬性 | 說明 |
|------|------|
| `character_name` | 名稱（如 "Boar"、"Raccoon"、"Fox"） |
| `gem_type` | 對應的寶石類型（決定哪種寶石觸發攻擊） |
| `level` | 等級 |
| `base_atk / atk_growth` | 基礎攻擊力與每級成長 |
| `base_hp / hp_growth` | 基礎血量與每級成長 |
| `passive_skill_name` | 被動技能名稱 |
| `active_skill_name / active_skill_cd` | 主動技能名稱與冷卻回合數 |
| `responding_skills` | 回應技能清單（陣列） |

**攻擊力公式**：`ATK = base_atk + floor(level × atk_growth)`
**血量公式**：`HP = base_hp + floor(level × hp_growth)`

### EnemyData（敵人資料）

| 屬性 | 說明 |
|------|------|
| `enemy_name` | 敵人名稱 |
| `max_hp` | 最大血量 |
| `attack_damage` | 每次攻擊傷害 |
| `attack_interval` | 每 N 回合攻擊一次 |
| `element` | 元素屬性（用於克制計算） |

## 5. 棋盤系統 (Board)

### 核心概念

棋盤是一個 `columns × rows` 的二維網格，每一格包含一個 `Block`（寶石）或 `null`（空格）。

### 初始化流程

1. `_ready()` 讀取關卡資料（`StageData`）
2. `_apply_stage()` 設定棋盤參數（大小、允許寶石、最小匹配數）
3. `initialize_board()` 為每一格建立隨機寶石

### 座標系統

- **網格座標** `Vector2i(x, y)`：x=欄位（左→右），y=行（上→下）
- **世界座標** `Vector2`：像素位置，每格 64×64 像素
- `grid_to_world()`：網格 → 世界（格子中心點）
- `world_to_grid()`：世界 → 網格

### 點擊消除流程

```
玩家點擊
    │
    ▼
_unhandled_input() 檢測滑鼠左鍵點擊
    │
    ▼
_handle_click(pos)
    │
    ├─ 是高階寶石？→ _handle_upper_click() → 特殊爆炸
    │
    ├─ _find_connected(pos) 用 BFS 找出所有相連的同色寶石
    │
    ├─ 連接數 < min_match？→ 播放抖動動畫，結束
    │
    └─ 連接數 ≥ min_match？
         │
         ├─ _destroy_blocks() 消除寶石（計分、發出信號、播放動畫）
         │
         ├─ 若 skip_collapse（融合流程）→ 暫停，等待 main.gd 處理
         │
         └─ _collapse_and_fill() 掉落並填充新寶石
```

### BFS 連接搜尋 (`_find_connected`)

從點擊位置出發，四方向（上下左右）搜尋相同類型的寶石：
- 忽略空格和不同類型
- 忽略高階寶石（不參與普通配對）
- 返回所有相連位置的陣列

### 掉落與填充 (`_collapse_and_fill`)

1. **壓縮**：每一欄從下往上掃描，將現有寶石向下移動填補空位
2. **填充**：空位從棋盤上方生成新寶石
3. **動畫**：所有移動以 `FALL_SPEED`（800 像素/秒）等速掉落 + 彈跳效果
4. 完成後更新融合提示

### 寶石類型

| 類型 | 顏色 | 元素 |
|------|------|------|
| RED | 紅色 | 火 |
| BLUE | 藍色 | 水 |
| GREEN | 綠色 | 葉 |
| YELLOW | 黃色 | — |
| PURPLE | 紫色 | — |
| ORANGE | 橙色 | — |

## 6. 戰鬥系統 (BattleManager)

### 職責

- 管理回合計數
- 管理敵人波次
- 計算攻擊傷害與元素克制
- 追蹤技能冷卻
- 處理敵人行動

### 玩家血量

玩家總血量 = 隊伍中所有角色的最大 HP 加總。

### 傷害計算

```
基礎傷害 = ATK × 消除寶石數量
最終傷害 = 基礎傷害 × 元素倍率
```

**元素克制**（1.5 倍）：
- 火(紅) → 葉(綠)
- 葉(綠) → 水(藍)  
- 水(藍) → 火(紅)

### 回合流程

```
玩家消除寶石
    │
    ▼
record_blast() 記錄本回合消除資料
    │
    ▼
get_attack_data() 計算攻擊傷害
    │
    ▼
check_responding_skills() 檢查回應技能觸發
    │
    ▼
finish_turn()
    ├─ 回合數 +1
    ├─ 冷卻遞減
    ├─ 清除本回合消除資料
    └─ 處理敵人行動
         ├─ 每個敵人檢查 turn % attack_interval == 0
         └─ 觸發攻擊：發出 enemy_attacked 信號
```

### 敵人波次管理

1. 每波有一組 `EnemyData`，生成對應的 `Enemy` 節點
2. 敵人全部消滅後等待 0.5 秒，進入下一波
3. 所有波次完成 → 發出 `battle_won` 信號
4. 第一個活著的敵人自動成為攻擊目標（可點擊切換）

## 7. 主場景控制器 (Main)

`main.gd` 是遊戲的核心控制器，協調所有子系統。

### 初始化 (`_ready`)

1. 從 GameState 讀取關卡與隊伍
2. 連接 Board 和 BattleManager 的信號
3. 設定角色面板
4. 初始化戰鬥
5. 設定融合提示

### 寶石消除後的處理 (`_on_gems_blasted`)

這是最核心的函式，根據情況分為兩種流程：

#### 融合流程（觸發了融合技能）

```
消除寶石
  │
  ▼
棋盤設定 skip_collapse = true（暫停掉落）
  │
  ▼
寶石粒子飛向點擊位置
  │
  ▼
在點擊位置放置高階寶石（火球/火柱）
  │
  ▼
執行掉落填充
  │
  ▼
結束回合
```

#### 普通攻擊流程

```
消除寶石
  │
  ▼
第1階段：寶石粒子飛向角色卡片
  │
  ▼
第2階段：角色攻擊序列
  │  ├─ 角色卡上彈
  │  ├─ 播放攻擊特效（斬擊/箭矢/粒子）
  │  ├─ 敵人受傷 + 傷害數字
  │  └─ 角色卡回位
  │
  ▼
第3階段：執行回應技能（如葉風暴）
  │
  ▼
第4階段：結束回合 → 敵人行動 → 回合開始被動技能
```

### 攻擊特效（依角色不同）

| 角色 | 元素 | 攻擊方式 |
|------|------|----------|
| Boar（野豬） | 水 | 斬擊特效 + 飲水被動（治療 50% 傷害） |
| Raccoon（浣熊） | 葉 | 每 3 個寶石射 1 箭，隨機命中存活敵人 |
| 其他 | — | 射出寶石色粒子到敵人 |

## 8. 技能系統

### 技能類型

#### 被動技能 (Passive)
- **自動觸發**，無需玩家操作
- 例：Boar 的「飲水」— 攻擊後治療 50% 傷害
- 例：Raccoon 的「光合作用」— 每回合開始轉 3 個寶石為葉

#### 主動技能 (Active)
- **玩家手動觸發**（點擊角色卡片）
- 有冷卻時間，每回合遞減
- 例：Boar 的「攻擊形態」— 將所有火寶石轉為水寶石
- 就緒時角色卡片發光提示

#### 回應技能 (Responding)
- **消除達標時自動觸發**
- 有門檻值（threshold）和觸發方式（count/line）
- 有優先級（數字越小越優先），同時觸發時只執行優先級最高的

| 技能名 | 角色 | 觸發條件 | 效果 |
|--------|------|----------|------|
| Leaf Storm | Raccoon | 消除 N+ 葉寶石 | 轉 3 個寶石為葉 |
| Fireball | Fox | 消除 N+ 火寶石 | 放置火球高階寶石 |
| Fire Pillar | Fox | 消除 N+ 火寶石連線 | 放置火柱高階寶石 |

### 融合提示系統

- 棋盤持續掃描所有連接群組
- 當某個群組達到融合門檻時，顯示閃爍的文字提示
- 依技能優先級排序，高優先級的提示覆蓋低優先級
- 每次棋盤變動後重新計算

## 9. 高階寶石系統

### 生成方式

當消除的同色寶石觸發回應技能（Fireball 或 Fire Pillar）時：
1. 棋盤暫停掉落
2. 寶石粒子飛向點擊位置
3. 在該位置創建高階寶石
4. 恢復掉落填充

### 高階寶石類型

| 類型 | 圖示 | 爆炸範圍 |
|------|------|----------|
| FIREBALL（火球） | 炸彈 | 3×3 核心 + 四方各延伸 1 格 |
| FIRE_PILLAR_X（橫火柱） | 旋風（旋轉90°） | 整行 |
| FIRE_PILLAR_Y（縱火柱） | 旋風 | 整欄 |

### 點擊高階寶石

1. 點擊高階寶石觸發爆炸
2. 爆炸範圍內的所有寶石被消除
3. **連鎖機制**：如果爆炸範圍內有其他高階寶石，會觸發連鎖爆炸
4. 每層連鎖增加 10% 攻擊力加成
5. 連鎖 ≥ 2 時顯示「Chain ×N!」動畫
6. 所有連鎖完成後統一計算傷害並播放攻擊動畫

### 連鎖爆炸流程

```
點擊高階寶石 A
    │
    ▼
計算 A 的爆炸範圍
    │
    ▼
消除範圍內所有寶石
    │
    ├─ 發現高階寶石 B → 加入連鎖佇列
    │
    ▼
處理連鎖佇列中的 B
    │
    ▼
計算 B 的爆炸範圍
    │
    ▼
消除範圍內未消除的寶石
    │
    ├─ 發現更多高階寶石 → 繼續連鎖
    │
    ▼
全部完成 → 統一計算傷害
```

## 10. 特效系統

所有特效都在 `FXLayer`（CanvasLayer）中生成，避免被棋盤遮擋。

### GemParticle（寶石粒子）
- 從消除的寶石位置飛向角色卡片
- 帶隨機偏移防止重疊
- 加速飛行效果

### SlashEffect（斬擊特效）
- Boar 攻擊時使用
- 18 幀精靈圖表逐幀播放
- 在動畫 50% 處觸發扣血

### BulletProjectile（子彈彈道）
- Raccoon 攻擊時使用
- 沿貝茲曲線弧形飛行
- 邊飛邊播放 8 幀動畫
- 精靈圖自動旋轉面向飛行方向

### Projectile（拋物線彈道）
- 敵人攻擊玩家時使用
- 從敵人飛向玩家血條
- 到達後觸發扣血回調

### DamageNumber（傷害數字）
- 五階段彈跳動畫
- 黑色描邊 + 陰影
- 克制效果時字體更大並加「!!」

## 12. 一次回合的完整流程

以下是一次**普通消除**的完整執行順序：

```
1. 玩家點擊棋盤上的寶石
   └─ board._unhandled_input() → board._handle_click()

2. BFS 搜尋相連寶石
   └─ board._find_connected() → 返回連接位置陣列

3. 檢查是否達到最小消除數
   └─ 不足 → 抖動動畫，結束
   └─ 足夠 → 繼續

4. 消除寶石
   └─ board._destroy_blocks()
       ├─ 設定 grid[x][y] = null
       ├─ 分數增加
       ├─ 發出 gems_blasted 信號
       └─ 播放消除動畫（非阻塞）

5. main._on_gems_blasted() 接收信號
   ├─ 記錄消除資料
   ├─ 檢查回應技能
   │
   ├─【若觸發融合技能】
   │   ├─ 暫停掉落
   │   ├─ 粒子飛向點擊位置
   │   ├─ 放置高階寶石
   │   ├─ 恢復掉落填充
   │   └─ 結束回合
   │
   └─【普通攻擊流程】
       ├─ 計算攻擊資料（傷害、目標、是否克制）
       ├─ 粒子飛向角色卡片
       ├─ 角色攻擊動畫序列
       │   ├─ 卡片上彈
       │   ├─ 特效 + 扣血 + 傷害數字
       │   └─ 卡片回位
       ├─ 執行回應技能（如葉風暴）
       └─ 結束回合

6. 結束回合
   └─ battle_manager.finish_turn()
       ├─ 回合數 +1
       ├─ 冷卻遞減
       ├─ 清除本回合資料
       └─ 處理敵人行動
           └─ 達到攻擊間隔的敵人 → 發射彈道 → 扣血

7. 回合開始被動
   └─ main._process_turn_start_passives()
       └─ 如 Raccoon 的光合作用：轉 3 寶石為葉

8. 更新技能 UI
   └─ main._update_skill_ui()
       └─ 更新冷卻數字、就緒發光

9. 棋盤掉落填充完成
   └─ board._collapse_and_fill()
       ├─ 現有寶石向下壓縮
       ├─ 新寶石從頂部掉落
       └─ 更新融合提示

10. 等待玩家下一次點擊...
```
