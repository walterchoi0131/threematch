# 角色 & 敵人數值成長設計提案 (Lv 1–99)

> **前提假設**：4 名角色 vs 1 隻敵人；每回合平均消除 ≈ 5 顆有效寶石  
> **核心約束**：等級差 +5 → 幾乎不可能獲勝；等級差 −5 → 輕鬆取勝

---

## 1. 成長公式

### 1.1 角色（單隻）

| 屬性 | 公式 | 說明 |
|------|------|------|
| **ATK(L)** | `floor(8 × 1.065^(L−1))` | 指數成長，每 5 級 ≈ +37% |
| **HP(L)**  | `floor(50 × 1.065^(L−1))` | 同上，HP 基數較高 |
| **MAG(L)** | `floor(6 × 1.065^(L−1))` | 魔力值。部分主動技傷害以 `MAG × 倍率` 計算（取代 ATK） |
| **累積 EXP(L→L+1)** | `floor(80 × L^1.5)` | 多項式成長，後期升級漸慢 |

> **與現有系統差異**：目前 `get_atk()` / `get_max_hp()` 為線性公式（`base + level × growth`）。  
> 建議改為指數公式以確保 5 級差距始終產生足夠的戰力落差。  
> 實作方式：在 `CharacterData` 加入 `atk_base_coeff` (8) 與 `growth_rate` (1.065) 等欄位。

### 1.2 敵人（單隻）

| 屬性 | 公式 | 設計目的 |
|------|------|----------|
| **HP(E)** | `floor(400 × 1.065^(E−1))` | = 50 × Char_ATK(E)，等級戰約 10 回合擊殺 |
| **ATK(E)/hit** | `floor(35 × 1.065^(E−1))` | = 0.7 × Char_HP(E)，等級戰承受 ≈ 65% 總 HP |
| **攻擊間隔** | 3 回合（一般）/ 1–2 回合（Boss） | Boss 壓力倍增 |
| **EXP 掉落** | `floor(25 × E^1.2)` | 擊殺同等級敵人 ≈ 7–9 隻升 1 級 |

### 1.3 關鍵倍率

| 等級差 | 倍率 `1.065^n` | 效果 |
|--------|----------------|------|
| ±1 級 | ×1.065 | 微幅差距 |
| ±3 級 | ×1.208 | 感覺得到差距 |
| ±5 級 | ×1.370 | 決定性差距 |
| ±10 級 | ×1.877 | 碾壓 / 被碾壓 |

## 3. 戰鬥模擬驗證

> 假設：每回合隊伍平均造成 `5 × Char_ATK` 傷害；敵人每 3 回合攻擊一次（回合 0、3、6…）

### 3.1 等級戰 — Lv 50 隊伍 vs Lv 50 敵人 ✅ 挑戰但可勝

| 項目 | 數值 |
|------|------|
| 隊伍 DPS/回合 | 5 × 175 = **875** |
| 擊殺回合數 | 8,755 / 875 ≈ **10 回合** |
| 敵人攻擊次數 | 回合 0, 3, 6, 9 = **4 次** |
| 總承受傷害 | 4 × 766 = **3,064** |
| 隊伍 HP | **4,376** |
| **HP 損耗率** | 3,064 / 4,376 = **70%** → 剩 30% HP 獲勝 |

### 3.2 越級挑戰 — Lv 50 隊伍 vs Lv 55 敵人 ❌ 幾乎不可能

| 項目 | 數值 |
|------|------|
| 敵人 HP | 8,755 × 1.37 ≈ **11,994** |
| 敵人 ATK | 766 × 1.37 ≈ **1,049** |
| 擊殺回合數 | 11,994 / 875 ≈ **14 回合** |
| 敵人攻擊次數 | 回合 0, 3, 6, 9, 12 = **5 次** |
| 總承受傷害 | 5 × 1,049 = **5,245** |
| 隊伍 HP | **4,376** |
| **HP 損耗率** | 5,245 / 4,376 = **120%** → ☠️ 隊伍全滅 |

### 3.3 降級碾壓 — Lv 50 隊伍 vs Lv 45 敵人 ✅ 輕鬆獲勝

| 項目 | 數值 |
|------|------|
| 敵人 HP | 8,755 × 0.73 ≈ **6,391** |
| 敵人 ATK | 766 × 0.73 ≈ **559** |
| 擊殺回合數 | 6,391 / 875 ≈ **8 回合** |
| 敵人攻擊次數 | 回合 0, 3, 6 = **3 次** |
| 總承受傷害 | 3 × 559 = **1,677** |
| 隊伍 HP | **4,376** |
| **HP 損耗率** | 1,677 / 4,376 = **38%** → 輕鬆過關 |

### 3.4 不同等級區間驗證

| 場景 | 隊伍 Lv | 敵人 Lv | 差距 | 擊殺回合 | HP 損耗 | 結果 |
|------|---------|---------|------|---------|---------|------|
| 新手等級戰 | 1 | 1 | 0 | 10 | 70% | ✅ 挑戰 |
| 新手越級 | 1 | 6 | +5 | 14 | 120% | ❌ 全滅 |
| 中期等級戰 | 25 | 25 | 0 | 10 | 70% | ✅ 挑戰 |
| 中期碾壓 | 25 | 20 | −5 | 8 | 38% | ✅ 輕鬆 |
| 後期等級戰 | 75 | 75 | 0 | 10 | 70% | ✅ 挑戰 |
| 後期越級 | 75 | 80 | +5 | 14 | 120% | ❌ 全滅 |
| 後期碾壓 | 99 | 94 | −5 | 8 | 38% | ✅ 輕鬆 |

> 由於使用指數成長，**5 級差距在任何等級區間都保持一致的 ×1.37 倍率**，確保平衡感從頭到尾不會崩壞。

## 5. Boss 設計建議

Boss 敵人使用相同的等級公式，但調整倍率：

| Boss 類型 | HP 倍率 | ATK 倍率 | 攻擊間隔 | 適用場景 |
|-----------|---------|---------|---------|---------|
| 精英怪    | ×1.5   | ×1.2    | 2 回合  | 每關卡最後一波 |
| 中 Boss   | ×2.5   | ×1.5    | 2 回合  | 章節中段 Boss |
| 大 Boss   | ×4.0   | ×1.8    | 1 回合  | 章節最終 Boss |

**Boss 模擬 — Lv 50 大 Boss（等級戰）**：
- HP = 8,755 × 4.0 = 35,020
- ATK = 766 × 1.8 = 1,379 / 每回合攻擊
- 擊殺回合 = 35,020 / 875 ≈ 40 回合
- 承受傷害 = 40 × 1,379 = 55,160 → 遠超隊伍 HP (4,376)
- → **必須搭配治療、護盾等技能才能存活**，符合 Boss 戰設計

## 6. 程式實作建議

### 6.1 CharacterData 修改

```gdscript
# 新增欄位
@export var atk_coeff: float = 8.0      # ATK 基礎係數
@export var hp_coeff: float = 50.0       # HP 基礎係數
@export var growth_rate: float = 1.065   # 每級成長率

func get_atk() -> int:
    return int(floor(atk_coeff * pow(growth_rate, level - 1)))

func get_max_hp() -> int:
    return int(floor(hp_coeff * pow(growth_rate, level - 1)))
```

### 6.2 EnemyData 新增等級系統

```gdscript
@export var enemy_level: int = 1
@export var hp_coeff: float = 400.0
@export var atk_coeff: float = 35.0
@export var growth_rate: float = 1.065
@export var exp_drop_base: float = 25.0  # EXP 掉落

func get_max_hp() -> int:
    return int(floor(hp_coeff * pow(growth_rate, enemy_level - 1)))

func get_attack_damage() -> int:
    return int(floor(atk_coeff * pow(growth_rate, enemy_level - 1)))

func get_exp_drop() -> int:
    return int(floor(exp_drop_base * pow(enemy_level, 1.2)))
```

### 6.3 GameState 新增 EXP 系統

```gdscript
var party_exp: int = 0
var party_level: int = 1

func exp_to_next_level() -> int:
    return int(floor(80.0 * pow(party_level, 1.5)))

func add_exp(amount: int) -> void:
    party_exp += amount
    while party_exp >= exp_to_next_level():
        party_exp -= exp_to_next_level()
        party_level += 1
        # 同步更新所有角色等級
        for c in selected_party:
            c.level = party_level
```

## 7. 設計摘要

| 設計目標 | 實現方式 |
|---------|---------|
| +5 級不可戰勝 | 指數成長 ×1.37/5 級 → 敵人輸出超過隊伍 HP |
| −5 級輕鬆過關 | 同上 → 敵人僅造成 ~38% HP 傷害 |
| 等級戰有挑戰 | 10 回合擊殺，承受 ~70% HP 傷害 |
| 升級節奏穩定 | 打同等級敵人 ~7-9 隻升 1 級 |
| 全等級平衡一致 | 指數公式確保任何等級區間的 5 級差距效果相同 |
| 低等級刷怪效率遞減 | 多項式 EXP 掉落 vs 多項式 EXP 需求 |
